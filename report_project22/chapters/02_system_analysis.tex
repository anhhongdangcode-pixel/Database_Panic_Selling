\section{Functional Requirements}

The system must support the following core functionalities:

\begin{enumerate}
    \item \textbf{Investor data management:} Store and manage investor profiles including psychological parameters (chasing bias, loss aversion, risk appetite, impatience).
    \item \textbf{Synthetic data generation:} Generate realistic market data and investor trading histories reflecting market stress scenarios.
    \item \textbf{Behavioral feature extraction:} Compute indicators including drawdown level, sell spike intensity, and loss sensitivity from transaction histories.
    \item \textbf{Panic score estimation:} Aggregate behavioral features into a normalized Panic Score (0.0 to 1.0) per investor.
    \item \textbf{Warning generation:} Automatically classify investors into Low, Medium, and High panic levels and generate advisory warnings.
    \item \textbf{Reporting:} Produce reports on score distributions, warning levels, and behavioral signals per investor.
\end{enumerate}

\section{Non-Functional Requirements}

\begin{itemize}
    \item \textbf{Performance:} Queries on large transaction tables (500k+ rows) must complete within acceptable time through proper indexing.
    \item \textbf{Scalability:} The schema should support extension to additional tickers and longer time horizons.
    \item \textbf{Maintainability:} All database logic encapsulated in stored procedures and functions for easy updates.
    \item \textbf{Security:} Role-based access control separating administrative, analytical, and read-only users.
\end{itemize}

The system consists of three main layers:

\begin{enumerate}
    \item \textbf{Data Layer:} MySQL database storing all investor, market, transaction, portfolio, behavioral signal, and warning data.
    \item \textbf{Analytics Layer:} Python scripts for data generation, feature engineering, and Panic Score computation.
    \item \textbf{Presentation Layer:} Flask web application providing an interactive interface for querying and monitoring investor behavior.
\end{enumerate}
